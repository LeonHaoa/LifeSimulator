# LifeSimulator UI 布局重构（方案 B）— 居中舞台 + 底部指令条 + 年鉴抽屉

**日期：** 2026-04-10  
**范围：** 仅 UI 布局与视觉层级（不改游戏规则、不改 API 协议、不改叙事逻辑）。  
**目标：** 让 `/life` 页面更像常见游戏界面：主体居中、舞台感更强、操作入口集中且稳定、信息分层清晰。

---

## 1. 现状问题（观察）

当前 `/life`（`src/components/LifeDetailClient.tsx` + `src/app/globals.css`）是「主面板 + 右侧年鉴」的网页应用布局：

- 主内容存在“网页感”：按钮与交互散落在面板内部，缺少“指令条”稳定位置。
- 右侧 `aside` 在大屏出现后形成典型 dashboard 两列；但“舞台中心”不够明确，视线会在属性卡、分配、叙事、年鉴之间跳。
- 年鉴本质是辅助信息，常见游戏里更符合“抽屉/叠层”模式，而不是常驻侧栏。

---

## 2. 设计原则

- **居中舞台**：主内容在视觉上居中并形成“主舞台框”，周围留出氛围背景空间。
- **指令条固定**：核心动作永远在底部指令条（Command Bar），不随内容高度变化。
- **辅助信息抽屉化**：年鉴/导出在抽屉里（Drawer），通过按钮打开；不抢主舞台空间。
- **单列优先**：默认单列居中；在超宽屏也不扩成左右常驻两列。
- **动效克制**：沿用现有渐变/发光语言；减少过多新动画，尊重 `prefers-reduced-motion`。

---

## 3. 信息架构与布局

### 3.1 顶部 HUD（保持，但更“游戏”）

顶栏维持在居中容器内（与舞台同宽），但更像 HUD：

- 左侧：标题、回合徽章
- 右侧：音效/音乐开关、返回
- 不承载主要操作（主要操作全部下沉到底部指令条）

### 3.2 居中舞台（Stage）

舞台为单列，包含三块：

1. **角色状态区**：属性卡网格（现有 `.stat-grid` 可用，但宽度随舞台收敛）
2. **本回合操作区**：技能分配（step=allocate）或转场（transition）或叙事（streaming/idle）
3. **里程碑 Banner**：保留

舞台宽度建议：

- `max-width`: `min(980px, 100vw - 2rem)`（比当前 1100 更收敛）
- 舞台上部与背景留白更明显

### 3.3 底部指令条（Command Bar）

固定在视口底部，居中，宽度与舞台一致：

- 左侧：当前模式提示（gain/lose/none）、剩余点数（或“无需分配”）
- 中间：主要行动按钮（开启下一年 / 下一年 / 先分配完点数等）
- 右侧：次要入口（年鉴按钮、导出按钮可进入抽屉）

指令条应：

- 有轻微玻璃拟态与边框发光
- 在 iPhone 安全区适配（`padding-bottom: env(safe-area-inset-bottom)`）

### 3.4 年鉴抽屉（Drawer）

抽屉默认关闭，点击指令条上的“年鉴”打开：

- 右侧滑入（desktop）或底部上拉（mobile）二选一：
  - 推荐：**底部上拉**（更“主机/手游”统一）；宽屏也可以底部上拉，减少平台差异。
- 抽屉内容：
  - 年鉴列表（`history-list`）
  - 导出 JSON（保留）
  - 可选：显示本局 seed/版本等（若已有，不新增）

抽屉交互：

- Esc 关闭（桌面）
- 点击遮罩关闭
- `aria-modal` / `role="dialog"`；焦点圈定（简化实现：只保证 Esc + 点击遮罩；不强制完整 focus trap）

---

## 4. 组件与代码改动范围

### 4.1 `src/components/LifeDetailClient.tsx`

- 将 `<aside>` 常驻年鉴移除，改为 Drawer 组件（仍在同文件内即可，避免拆分过多）。
- 把“开启下一年 / 下一年”等按钮从面板内部迁移到固定底部指令条：
  - `step === "allocate"`：指令条展示“开启下一年”主按钮
  - `step === "idle"`：指令条展示“下一年”主按钮
  - `step === "transition" | "streaming"`：主按钮 disabled + 显示状态
- 在 state 中增加：`isJournalOpen: boolean`（抽屉开关）

### 4.2 `src/app/globals.css`

新增/调整样式（命名建议）：

- `.life-shell`：页面整体（替代或包裹 `.life-page`）
- `.life-stage`：居中舞台容器
- `.life-commandbar` / `.life-commandbar__inner`：底部指令条
- `.life-drawer` / `.life-drawer__overlay` / `.life-drawer__panel`：抽屉

并调整：

- `.life-page` 的 `max-width` 与 padding：改为“舞台 + 指令条安全区”
- `.life-grid`：不再使用两列布局（删除 `@media (min-width: 860px)` 的侧栏结构或将其改为单列）

---

## 5. 响应式与细节规范

- **<= 860px**：舞台全宽（留边距），指令条高度更高（按钮更大），抽屉底部上拉占 70–85% 高度。
- **> 860px**：舞台保持 max-width 收敛；抽屉仍底部上拉（或右侧滑入需额外适配，P0 先统一底部）。
- **滚动**：页面内容可滚动，但指令条固定；需要 `padding-bottom` 预留指令条高度，避免内容被遮挡。
- **可访问性**：按钮 `aria-pressed` 现有即可；抽屉 `role="dialog" aria-label="年鉴"`；overlay 可点击关闭。

---

## 6. 验收标准（Definition of Done）

在 `npm run dev:stable` 下：

- `/life` 主舞台在所有常见尺寸（手机竖屏、iPad、桌面 1440）都**明显居中**，两侧有氛围留白。
- “开启下一年 / 下一年”始终在底部指令条，且不随面板高度跳动。
- 年鉴不再常驻侧栏；通过指令条按钮打开/关闭，动画与遮罩正常。
- 不引入新的 ESLint error；`npm run lint` 与 `npm run build` 通过。
