## 2. 范围与假设

- **单局状态**：浏览器持有完整 `GameState`，每次「下一年」请求携带 state（与闭环规格 §1.4 一致）。
- **刷新**：刷新页面丢失当前局（README 需写明）；用户可通过 **导出 JSON** 自行备份截屏外的「状态快照」。
- **导出（B）**：提供「导出存档」动作，下载包含完整可序列化状态与元数据的 JSON 文件；**不实现导入**，避免 Zod 校验、版本迁移与恶意 payload 在第一版分散精力。
- **登录 / KV / D1**：不在本设计 P0/P1 范围内。

## 4. 导出 JSON（B）契约

**文件名建议：** `life-simulator-save.json`（实现可调整，README 写明）。

**顶层字段（最小约定）：**

- `schemaVersion`：与运行时 state 版本一致，与 API 校验共用。
- `exportedAt`：导出时刻 ISO 8601 字符串。
- `game`：与 `POST /api/year` 使用的 `GameState` 同构对象（含 `runSeed`、姓名、年龄、属性、累计标签或历史等实现定义字段）。

**显式非目标：** 第一版不提供「从文件恢复」按钮；导出仅供用户自备备份与向面试官展示可复现数据。

---

## 5. 模块与数据流

- **客户端**：持有 `GameState`；发起 `/api/year`；可选消费流式叙事；提供导出（纯前端序列化即可，无需额外 API）。
- **事件引擎包（可选路径 `packages/engine`）**：`advanceYear(state, rng)` → 事件列表与属性补丁；禁止 Node 专有 API 与网络。
- **Route Handler**：校验 → 引擎 → 叙事模块（非流式或流式分支）；密钥仅服务端。
- **静态事件数据**：构建期 `import`，Worker 运行时不用 `fs`。

**流式与状态一致性：** 若采用「先展示流式叙事再提交 state」，须保证中断时不会把不完整叙事与已递增年龄写死在同一不可恢复状态；具体策略在实现计划中写死（例如：流式仅展示，成功结束后单次提交；或叙事失败回滚 UI 状态）。

---

## 6. 非功能验收（与闭环自检对齐）

1. 断网或模型不可用：仍可推进年份（降级叙事）。  
2. 无效或缺失 API Key：接口与 UI 仍可完成闭环。  
3. 叙事超时：不白屏、不重复加岁。  
4. Cloudflare：构建与生产非流式必过；若做流式则生产域端到端验证。

## 7. 设计自检（spec self-review）

- **占位符：** 无 TBD；工程子路径在实现计划中细化。
