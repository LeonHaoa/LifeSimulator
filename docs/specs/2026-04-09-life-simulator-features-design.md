## 2. 范围与假设

- **单局状态**：浏览器持有完整 `GameState`，每次「下一年」请求携带 state（与闭环规格 §1.4 一致）。
- **刷新**：刷新页面丢失当前局（README 需写明）；用户可通过 **导出 JSON** 自行备份截屏外的「状态快照」。
- **导出（B）**：提供「导出存档」动作，下载包含完整可序列化状态与元数据的 JSON 文件；**不实现导入**，避免 Zod 校验、版本迁移与恶意 payload 在第一版分散精力。
- **登录 / KV / D1**：不在本设计 P0/P1 范围内。

## 4. 导出 JSON（B）契约

**文件名建议：** `life-simulator-save.json`（实现可调整，README 写明）。

**顶层字段（最小约定）：**

- `schemaVersion`：与运行时 state 版本一致，与 API 校验共用。
- `exportedAt`：导出时刻 ISO 8601 字符串。
- `game`：与 `POST /api/year` 使用的 `GameState` 同构对象（含 `runSeed`、姓名、年龄、属性、累计标签或历史等实现定义字段）。

**显式非目标：** 第一版不提供「从文件恢复」按钮；导出仅供用户自备备份与向面试官展示可复现数据。

## 5. 模块与数据流

- **客户端**：持有 `GameState`；发起 `/api/year`；可选消费流式叙事；提供导出（纯前端序列化即可，无需额外 API）。
- **事件引擎包（可选路径 `packages/engine`）**：`advanceYear(state, rng)` → 事件列表与属性补丁；禁止 Node 专有 API 与网络。
- **Route Handler**：校验 → 引擎 → 叙事模块（非流式或流式分支）；密钥仅服务端。
- **静态事件数据**：构建期 `import`，Worker 运行时不用 `fs`。

**流式与状态一致性（已定：策略 S1）：**

- **S1：** 流式（未来启用时）**仅影响展示**；**已提交的 `GameState` 不得出现「半段叙事」**。任一回合在**客户端视角**须 **原子**：要么仍停留在点击「下一年」**之前**的快照，要么一次性收到并应用**完整**新状态（年龄 +1 一次、事件列表、叙事定稿或降级文案）。  
- **半道断开：** 丢弃未完成流式展示，**不**把半成品 narrative 写入持久状态；**禁止**在同一回合内再次 **+1 岁**（与下文「禁止重复加岁」一致）。

## 6. 非功能验收与错误语义（已定）

以下为闭环自检条目的**可测试定义**，与产品确认一致。

### 6.1 断网

- **要求：** 用户设备与 LLM 网络不通时，仍可完成「下一年」（事件引擎 + **本地/模板叙事**）。  
- **已定：** **断网能玩。**

### 6.2 无效 / 缺失 API Key（方案 A，全链路统一）

- **含义说明：** 「200」指 **HTTP 状态码 200**（成功响应），而不是返回 401/500 让前端进错误页。服务端仍返回**合法 JSON**，其中叙事字段为 **降级文案**（模板拼接），**不**调用外网模型。  
- **已定：** **统一采用方案 A（推荐 Demo）** —— Key 缺失、明显无效、或模型不可用：**仍 HTTP 200** + 完整游戏 payload + **本地叙事**；与断网、超时最终失败等**对外表现一致**（便于评委一条路径录通）。

### 6.3 叙事超时与重试（禁止重复加岁）

- **含义说明：**  
  - **不白屏：** 超时或重试中须有加载/重试提示或降级路径，不得空白卡死。  
  - **不重复加岁：** 用户点一次「下一年」，**年龄最多 +1**；禁止因超时重试、重复请求、流式重连导致 **+2 岁及以上**。  
- **已定策略：**  
  1. **尽量**在本回合内保留**已算出的引擎结果**（服务端一次请求内先引擎后叙事，或客户端缓存本回合 `engineResult` 仅用于重试叙事 —— 实现计划二选一写死）。  
  2. **重试：** 对 LLM 调用做 **有限次重试**（含退避）；仍失败则 **降级文案**，且本回合仍只算 **一岁**。  
  3. **草稿丢失：** 若用于重试的暂存丢失，则 **视为本回合未成功提交**：**不上报**不完整状态为「已推进」；UI 回到**推进前节点**（与点击「下一年」前一致）。  
  4. **整页刷新：** 与「无云存档」一致 —— 未导出则进度可能丢失；若实现使用 `sessionStorage` 仅保存**上一已提交快照**用于防半成品，须在 README 写明行为边界。

### 6.4 流式半道断开（策略 S1）

- **含义说明：** 避免「流式输出到一半刷新/断线 → 年龄已加、叙事半截、无法恢复」。  
- **已定：** **S1** —— 见 §5「流式与状态一致性」：半段**不**写入已提交状态；断开回到**本回合前**快照；**禁止重复加岁**。

### 6.5 Cloudflare 与流式分期

- **已定：** 先做 **非流式** 生产可用；流式 **预留接口** + README 说明后续在 CF 实测（与 F1.1 一致）。
