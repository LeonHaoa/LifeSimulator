# 中式人生模拟器 Demo — 程序闭环、架构与随机种子规格

面向 **3 天级交付** 与 **面试评审**（0→1 跑通、系统设计、可扩展/鲁棒、多轮与流式 AI）。**目标部署：Cloudflare Pages（Workers 运行时）** —— 技术选型与 API 实现须以该运行时为前提。本文档合并：**主 Prompt**、**评判映射**、**前后端语言与仓库策略**、**随机种子设计**、**Cloudflare 约束**。

**与功能设计的关系：** 已落地的交互与错误语义以 **`docs/specs/2026-04-09-life-simulator-features-design.md`** 为准（含：断网可玩、**方案 A** 缺 Key 仍 HTTP 200 + 降级叙事、超时重试与 **禁止重复加岁**、流式 **S1**、无限年 + 70/80/90/100 提示、先非流式 + 预留流式、LLM 输入仅事件 id 等）。本文主 Prompt 已尽量对齐；若有歧义，以前述功能设计为准。

## 1. 前后端语言与架构建议

### 1.1 推荐组合（默认，与题目技术栈一致）

| 层级 | 语言 / 框架 | 说明 |
|------|-------------|------|
| 前端 | **TypeScript + React**（**Next.js App Router**） | 与题目一致；SSR/Route Handler 统一单仓；**按 Cloudflare 适配方案构建**（见下文 §1.4）。 |
| 服务端 | **TypeScript**，运行于 **Workers（边缘）** | 通过 Next 的 Route Handlers / 适配后的 Server 片段对外提供 API；**不是**传统常驻 Node 进程。 |
| 数据 / 配置 | **TS 常量表或构建期导入的 JSON + Zod 校验** | 事件池打进 bundle 或静态资源，**避免在 Worker 内依赖 Node `fs` 读盘**（见 §1.4）。 |

**架构要点（单仓单体）：**

- **领域层**：人物状态、年龄、属性、`schemaVersion`。
- **事件引擎**：纯函数 `(state, year, rng) → events`，**零网络**、可单测。
- **叙事层**：独立模块，调用 LLM；失败走降级模板；可选 SSE 流式输出。

### 1.2 可选组合（仅当你要刻意展示「双栈」）

- 前端：仍建议 **Next.js + TS**（评委熟悉）。
- 后端：**Python（FastAPI）** 单独服务，负责事件引擎 + LLM 代理；前端通过 `fetch` / SSE 调用。
- **代价**：CORS、两套依赖、两套部署与环境变量；3 天内维护成本明显高于单体 Next。

### 1.3 要不要分成 2 个仓库？

**默认建议：不要。** 使用 **单仓库（monorepo）** 即可，例如：

```text
apps/web/          # Next.js 全栈（页面 + API + SSE）
packages/engine/   # 可选：共享「事件引擎」包，供单测与 API 引用
packages/shared/   # 可选：共享类型与 Zod schema
```

**仅在以下情况考虑双仓库：**

- 团队规范强制前后端分离且你已有模板 CI；
- 你要投递「微服务 + 独立扩缩容」叙事，并接受双倍运维叙述成本。

**面试沟通话术：** 「Demo 阶段用单仓降低摩擦；引擎与叙事边界在代码层已切开，后续可抽成独立 npm 包或 Python 服务而不改领域模型。」

### 1.4 目标部署：Cloudflare — 规划与选型约束（必读）

**部署形态（规划）：** 单仓库 → **Cloudflare Pages** 连接 Git，使用与当前 **Next.js 版本匹配的 Cloudflare 官方/推荐适配链路**（例如基于 **OpenNext 的 Cloudflare 方案** 或文档当前推荐的 Next-on-Workers 构建方式）。具体包名与命令以 **Next 主版本 + Cloudflare 文档** 为准，本规格只约束**能力与禁忌**，避免因绑定过时包名而误导。

**运行时本质：** 请求处理跑在 **Workers** 上，API 与 SSR 须符合 **WinterTC / Web 标准** 心智，而非「随便用 Node 标准库」。

| 类别 | 建议 | 原因 |
|------|------|------|
| **HTTP / AI** | 用 **`fetch`** 调 LLM；超时用 `AbortSignal` | Workers 一等公民；避免依赖仅 Node 可用的 HTTP 客户端 |
| **流式叙事** | **`Response` + `ReadableStream`**（SSE 或 NDJSON） | 与 Workers 兼容；**必须在真实 CF 预览环境测一遍** |
| **加解密 / 哈希** | **`crypto.subtle` / Web Crypto**（种子哈希、可选签名） | 避免 Node `crypto` 在边缘路径上的兼容坑 |
| **事件数据** | **import 静态 JSON / TS 常量**，或构建步生成内联 | 运行时**不要** `fs.readFile`、不要假设本地磁盘 |
| **依赖选型** | 优先 **纯 JS / 无原生扩展**；读包说明是否支持 Edge | 原生 `.node` 模块常在 Workers 不可用 |
| **重 CPU** | 事件引擎保持轻量；避免巨型同步循环阻塞 | Worker CPU 时间有限；大数据预处理放构建期 |
| **状态** | Demo 级可 **无 DB**：state 客户端持有 + POST 回传；若上 KV/D1 再单独设计 | 减少边缘存储与一致性叙述成本 |
| **密钥** | `OPENAI_API_KEY` 等仅 **Cloudflare 环境变量 / Secrets**，不进 Git | 与 Vercel 同理 |

**验收（部署相关，写入 README）：**

1. **生产构建**在 Cloudflare 流水线成功（非仅本地 `next dev`）。  
2. **至少一条路径**：非流式 `POST /api/year` 在 CF 上可用。  
3. **若承诺流式**：在 CF 预览/生产域名上实测 SSE 或 NDJSON 端到端；失败则文档写明降级策略。  
4. 文档中写明 **Node 版本**（本地）、**构建命令**、**输出绑定方式**（Pages 项目设置），方便评委复现。

**与「Python 后端」方案的关系：** 若后端独立 Python，则 Cloudflare 只托管静态前端 + 反代，**AI 与引擎仍在 Node/Python  VPS 或其他平台**；与本规格「单仓 Next 上 CF」不同，面试叙述要自洽，3 天成本更高。

## 2. 随机种子设计（闭环前提下更有趣）

目标：**同一输入可复现（便于调试与演示），不同种子体验分化明显**，且不改变「起名 → 属性 → 逐年推进 → 叙事」的闭环。

### 2.1 种子层级

| 种子 | 用途 | 建议生成方式 |
|------|------|----------------|
| **runSeed**（命运种子） | 整局 RNG 根；控制属性 roll、事件池排序、稀有事件门控 | 默认：`hash(规范化名字) ^ optionalUserSalt`；可选 UI「换一批人生」按钮生成新 `runSeed`。 |
| **yearSeed(y)** | 第 y 年子 RNG，避免「每年同一随机序列」 | `derive(runSeed, year)`，如 `hash32(runSeed, year)` 或 HMAC 取前 8 字节；**每年独立洗牌**。 |
| **eventDrawSeed** | 当年多条事件顺序与是否合并梗 | 从 `yearSeed` 再派生一条链（如 PCG / mulberry32）。 |
| **narrativeJitter** | 同事件组合下叙事口吻微差（非必须） | `hash(runSeed, year, eventIdsJoined)` 取少量枚举（冷面吐槽 / 温情损 / 互联网梗浓度）。 |

**约束：** 所有随机入口只通过 **注入式 RNG**（如 `mulberry32(seed)`、`seedrandom` 类库或自写 PRNG），**禁止**在引擎内 `Math.random()` 无种子，否则无法复现 bug。

### 2.2 与闭环、鲁棒的结合

- **状态里持久化**：`runSeed`、`currentYear`、`schemaVersion`；可选存 `rngCheckpoint`（一般不存完整 RNG 状态，用确定性派生即可）。
- **回放**：同一 `runSeed` + 相同决策序列 → 相同人生轨迹（若存在用户分支选项，则分支 id 须进入派生链）。
- **保底事件**：当筛选池为空时，用 `yearSeed` 固定抽取保底池，避免「全图无事件」卡死。
- **AI 不参与抽签**：种子只影响 **规则引擎与模板槽位**；LLM 仅润色，避免因模型温度导致「同一 seed 叙事完全不可比」（叙事可选 `temperature` 低或固定 seed 给模型若供应商支持）。

### 2.3 趣味性增强（仍属确定性层）

- **分层事件表**：普通 / 稀有 / 传说档，`yearSeed` 决定当年是否「解锁传说档一次」。
- **连锁 tag**：上年事件 tag 影响下年权重（引擎内状态可记 `recentTags[]`）。
- **名字彩蛋**：`hash(name)` 命中少量 id 固定事件（面试亮点，易演示）。

## 3. 评判标准与闭环映射

| 评判维度 | 闭环含义 | 评审时看什么 |
|----------|----------|--------------|
| **0. 0→1** | 起名 → 属性 → 逐年推进 → 可见反馈 | 一条路径录屏；无 AI 也能玩通 |
| **1. 系统设计 + AI 架构** | 引擎与 LLM 解耦、API 清晰 | README 数据流；引擎无网络；密钥仅服务端 |
| **2. 扩展 / 鲁棒 / AI 边界** | 新事件低耦合；异常有路径 | 超时、Zod 校验、降级、空池保底；`schemaVersion` |
| **3. 多轮 + AI 优化** | 长会话正确；流式体感 | 50+ 年点击；SSE/Stream + 非流式 fallback |

**自检清单：**（逐项定义见功能设计 **`docs/specs/2026-04-09-life-simulator-features-design.md` §6**）

1. 断网能否玩（叙事降级）？ — **已定：能。**  
2. 无效/缺失 API Key 是否仍 **HTTP 200** + 本地叙事？ — **已定：方案 A，是**（非 401/500 挡整条链路）。  
3. 叙事超时是否不白屏、不重复加岁？ — **已定：须加载/降级；禁止重复加岁；可重试；草稿丢失则回退推进前节点。**  
4. 流式半道断开是否不把半段写入不可恢复 state？ — **已定：策略 S1**（半段不入库；回退快照；不重复加岁）。

## 4. 主 Prompt（给 AI 编程助手 / 自研清单）

```markdown
你是资深全栈工程师。请在约定技术栈内实现「中式人生模拟器」Web Demo，目标不是堆功能，而是**可演示的完整闭环 + 清晰架构 + AI 流程可控**。

### 0. 必须跑通的 0→1 闭环（验收底线）
1. 用户输入名字 → 根据 **runSeed**（见规格文档「随机种子」）生成初始属性（至少 4 维，如颜值/家境/体质/运气等）；README 说明种子如何生成与复现。
2. 用户点击「下一年」→ 系统根据 **当前属性、年龄、yearSeed(year)** 从事件池筛选候选 → 加权/规则抽取本年 1～3 条事件；**禁止在引擎内直接调 LLM**。
3. 每条年度结果在 UI 上可滚动/逐条展示，支持「自动播放 / 手动下一步」至少一种明确模式。
4. 单会话内连续推进 ≥30 年无崩溃；建议压测 ≥50 年。刷新可丢会话，但文档说明行为。

### 1. 系统设计与 AI 架构（硬性要求）
**分层：**
- **领域层**：人物状态（immutable 更新）、年龄递增、事件 Schema（id、tags、条件谓词、权重、模板文案）。
- **事件引擎**：纯函数或可单测模块——输入 (state, year, rng) → 输出 `selectedEvents[]`，零网络。
- **叙事层（AI）**：输入结构化摘要（年份、昵称、属性摘要、本年事件 id/title/标签）；输出吐槽风中文短段；**失败必须降级**；README 说明降级策略。

**API：**
- `POST /api/year`（或等价）：body 含 state + 可选 `stream: boolean`；返回新 state + 本年事件 + 叙事（或流）。
- AI Key **仅服务端**。

### 2. 可扩展性、鲁棒性、AI 边界稳定
- 新事件以数据文件扩展为主，核心抽签逻辑稳定；条件可扩展（tag 交集、数值阈值）。
- LLM：timeout（如 8～15s）、有限重试、错误分类；响应 **Zod 校验**；失败降级。
- 输入校验：名字长度、非法字符、state `schemaVersion`。
- 空事件池：**保底事件** + `fallback: true` 可观测；**无强制寿命上限**（可持续推进直至用户新开一局）；**70 / 80 / 90 / 100** 岁里程碑文案与超时重试数值见功能设计 **§6.6～§6.8**。

### 3. 多轮无 bug 与 AI 优化（加分硬指标）
- 多轮：年龄单调、无重复扣减；双击防抖或 `requestId` 幂等（可选）。
- **流式：** 第一版可 **仅非流式**；须 **预留** `stream` / 流式端点及类型；README 写明后续在 Cloudflare 上补 SSE 或 NDJSON 并实测。若已启用流式：须满足 **S1**（半段不写入已提交 state、断开回退、不重复加岁）。

### 技术栈与部署目标
优先 **TypeScript + Next.js（App Router）** 单仓；引擎可抽到 `packages/engine`。**生产部署目标为 Cloudflare Pages（Workers）**：服务端逻辑禁止使用与 Workers 不兼容的 Node 专有 API；事件数据构建期内联；LLM 用 `fetch` + `AbortSignal`；**先保证非流式在生产可用**；流式启用后用 `ReadableStream` 并在 CF 实测（与功能设计分期一致）。

### 交付物
- GitHub：README（架构图、环境变量、种子与复现说明、**Cloudflare 构建与绑定步骤**）、`.env.example`。
- 单测或最小脚本：事件筛选、年度递增、LLM 失败降级、**同 runSeed 复现同一前 N 年**（可选但加分）。

### 实现顺序（强制）
1) 种子 + 无 AI 闭环 → 2) 引擎单测 → 3) AI + 降级 → 4) SSE → 5) UI 打磨。

开始前先输出：目录结构草案 + state/event 的 TS 类型 + API 契约 + runSeed/yearSeed 派生公式。
```

## 5. 给评委的一句话摘要

> **随机与玩法由确定性种子 + 事件引擎保证可复现与可测**；**LLM 仅叙事层且可降级**；**年度 API 契约稳定**；弱网/缺 Key 走 **HTTP 200 + 降级叙事（方案 A）**；**先非流式上线，预留 SSE/流式** 与 **S1** 状态边界，支撑多轮无 bug 会话。默认 **单仓 Next 全栈**；**生产部署以 Cloudflare Workers 为约束做选型**。
