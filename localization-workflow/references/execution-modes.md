# Execution Modes

## one-pass

Use when the profile is already stable and the goal is minimum interruption.

Flow:

1. freeze rules once
2. rollout
3. polish
4. validate
5. summarize at the end

## three-phase

Recommended default.

Flow:

1. phase 1: freeze profile and tone
2. phase 2: structure, parity, normalization
3. phase 3: polish, validation, acceptance summary

## release-polish

Use for existing projects that only need release-stage cleanup.

Flow:

1. skip structural rebuild
2. polish visible copy
3. validate
4. produce release summary
